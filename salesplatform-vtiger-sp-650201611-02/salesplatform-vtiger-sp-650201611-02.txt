SalesPlatform Vtiger CRM 6.5.0-201611 Service Pack 02
    http://community.salesplatform.ru

Содержимое пакета обновления:
* Доработана поддержка PHP 7.0, 7.1 (теперь поддерживаемыми версиями php являются 5.4.x - 7.1.x)
* В Журнал добавлена история изменений товарной части товарных модулей
* Исправлена ошибка отображения поля "Общий итог" в товарных модулях в режиме детального просмотра
* Обновлен PHPMailer - устранена уязвимость, позволяющая инициировать удаленное выполнение кода
* Удалена интеграция с социальной сетью Facebook (поле Facebook URL оставлено)
* Реализован механизм контроля сохранения карточки (вывод сообщения об ошибке вместо "белого экрана")
* PBXManager: улучшено управление показом уведомления о звонке только для активной вкладки
* Доработана локализация

Краткая инструкция по установке пакета обновления:
1. Перейдите в директорию, где развернута CRM (например, /var/www/vtigercrm).
2. Сделайте резервную копию файлов CRM и базы данных.
3. Скопируйте в директорию CRM архив с пакетом обновления. Для установки патча могут потребоваться права администратора.
4. Распакуйте архив:
    # tar xzf salesplatform-vtiger-sp-650201611-02.tar.gz
Архив содержит файлы с расширениями:
    patch - изменения исходного кода
    txt - описание пакета обновления и краткая инструкция.
5. Выполните команду проверки совместимости патча:
    # patch --dry-run -p 1 < salesplatform-vtiger-sp-650201611-02.patch
Вывод команды патч должен быть следующего вида:
    patching file somedir1/somefile1.php
    patching file somedir2/somefile2.php
Внимание! Если результат команды содержит строки вида Hunk #1 FAILED, не продолжайте обновление, если не являетесь разработчиком Vtiger. 
Среди возможных причин: не были установлены предыдущие пакеты обновления; исходный код системы изменен; 
версия системы не соответствует версии, для которой создан данный пакет обновления.
6. Выполните команду применения патча:
    # patch -p 1 < salesplatform-vtiger-sp-650201611-02.patch
